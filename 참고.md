# github-issue-manager 진행 정리

## 현재 결론

`codex-skills/github-issue-manager`는 **gh CLI 기반 GitHub issue 관리 MVP 스킬**로 정리되었다.

## 구현된 것

- GitHub 작업 기준을 MCP가 아니라 `gh` CLI로 통일
- `gh_readiness.py`로 `gh` 설치·인증 상태를 코드에서 강제 검사
- `template_mode.py`로 대상 레포의 `.github/ISSUE_TEMPLATE` 존재와 사용 가능 여부 자동 판단
- `gh_issue.py`로 issue `inspect`와 `create` 흐름 표준화
- issue 생성은 기본 dry-run, `--apply` 명시 시에만 실제 생성
- label 존재 여부 사전 검사
- branch naming은 `.github/github-issue-manager.yml` 설정 문서로 커스터마이즈 가능하게 정리
- 루트 `README.md`에 사용법을 간결하게 추가

## helper 역할

### `gh_readiness.py`

- `gh` CLI 존재 여부 확인
- `gh auth status` 인증 여부 확인
- 실패 시 JSON reason/remediation 출력 후 non-zero 종료

### `template_mode.py`

- 대상 레포의 `.github/ISSUE_TEMPLATE/`와 `.github/ISSUE_TEMPLATE.md` 검사
- usable Markdown template 또는 기본 필드가 있는 YAML issue form이면 `enabled: true`
- 없거나 불완전하면 `enabled: false`

### `gh_issue.py`

- `inspect`: readiness, repo 정보, template mode, label 상태 확인
- `create`: issue 생성 preview 출력
- `--apply`: 실제 `gh issue create` 실행 후 생성된 issue 재조회로 검증

## 의도적으로 하지 않은 것

- `branch_config.py`는 만들지 않음
  - 현재는 문서화된 설정 스키마만으로 충분함
  - branch 생성 자동화가 반복적으로 필요해질 때 코드화하는 편이 낫다
- issue 삭제, project, sub-issue, milestone 고급 자동화는 아직 helper로 만들지 않음
  - 직접 `gh` 또는 `gh api`로 처리하도록 SKILL.md에 남김

## 검증된 것

```bash
python3 /Users/seoho/.codex/skills/.system/skill-creator/scripts/quick_validate.py codex-skills/github-issue-manager
python3 -m py_compile codex-skills/github-issue-manager/scripts/*.py
python3 codex-skills/github-issue-manager/scripts/gh_issue.py inspect --repo nogie-dev/my-skills --repo-dir .
python3 codex-skills/github-issue-manager/scripts/gh_issue.py create --repo nogie-dev/my-skills --repo-dir . --title "Dry run validation" --body-file /tmp/issue.md
```

결과:

- Skill validation 통과
- Python syntax 검사 통과
- `gh` 인증된 계정으로 repo inspect 성공
- issue create dry-run 성공
- 실제 issue 생성은 `--apply`를 쓰지 않아 수행하지 않음

## 남은 확장 후보

- branch 이름 생성·충돌 검사 helper
- assignee/milestone 사전 검증
- issue body template 생성 helper
- label 정책 reference 추가
- 실제 `--apply` 생성 테스트용 disposable repo 시나리오
